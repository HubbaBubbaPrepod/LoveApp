1. Общее описание
Раздел «Художества» предназначен для совместного творчества в реальном времени между мной и моей девушкой. Мы оба используем приложение на своих Android-устройствах. Основная фишка — возможность рисовать одновременно на общем холсте, видя линии партнёра по мере их появления. Также требуется уникальная функция: рисование прямо на экране блокировки, где сам экран блокировки становится холстом, и все штрихи мгновенно отображаются у партнёра на его экране блокировки (или в приложении). На lock screen доступны все основные цвета.

Ключевые требования:

Синхронизация в реальном времени через WebSocket (Socket.IO) на собственном сервере (Node.js + Express), который уже работает.

Возможность создавать несколько холстов (картин). Холсты привязаны к паре (couple_key), создаются через существующее REST API.

Каждый холст отображается в виде плитки на главном экране раздела с миниатюрой, загружаемой через REST.

При открытии холста — полноэкранный редактор с инструментами (цвета, размер кисти, ластик, undo/redo). Рисование синхронизируется через WebSocket.

Виджет на главном экране Android (3 размера, как в других разделах), показывающий последний рисунок, отправленный партнёром (миниатюра из БД).

Экран блокировки как холст: интеграция с Android lock screen для рисования поверх заблокированного экрана с синхронизацией через WebSocket.

Детали интерфейса и функциональности
3.1. Главный экран раздела (Art Gallery)
Сетка плиток (2 колонки) с использованием LazyVerticalGrid (как в других разделах).

Каждая плитка (ArtTile) содержит:

Миниатюру рисунка (загружается через Coil из thumbnail_url), которая обновляется при изменениях холста.

Название холста (редактируется по долгому нажатию или через отдельный экран).

При нажатии открывается полноэкранный редактор с этим холстом.

Кнопка «+» для создания нового холста (вызывает POST /api/art/canvases, затем открывает редактор).

3.2. Экран рисования (Canvas Editor)
Полноэкранный холст с панелью инструментов (может скрываться):

Выбор цвета из фиксированной палитры (12 цветов, включая чёрный, белый, красный, синий и т.д.).

Ползунок размера кисти.

Переключатель ластика.

Кнопки Undo/Redo (локальные, не синхронизируются).

Кнопка «Назад» (сохраняет текущее состояние через загрузку миниатюры на сервер и выходит).

Рисование пальцем: каждое касание (начало, движение, окончание) преобразуется в структуру DrawingAction и отправляется через WebSocket в комнату пары.

При получении действия от сервера оно применяется к локальному холсту, отрисовывая линию партнёра (можно отображать другим цветом или с полупрозрачностью).

Холст имеет белый фон. Все штрихи хранятся локально в памяти (список действий) для поддержки Undo/Redo, но не сохраняются в БД как действия. Периодически (раз в 5–10 секунд или при выходе) клиент отправляет итоговый снимок холста (PNG) на сервер через POST /api/art/canvases/:id/thumbnail, чтобы обновить миниатюру и иметь постоянное хранилище.

3.3. WebSocket-события (Socket.IO)
Подключение: клиент отправляет JWT-токен (можно через handshake или отдельное событие authenticate). Сервер валидирует токен, получает userId и определяет couple_key (из таблицы relationship_info или из art_canvases). Добавляет клиент в комнату couple:couple_key.

События:

draw-action — отправляется клиентом при каждом значимом действии (начало штриха, движение, конец, очистка холста). Содержит:

json
{
  "canvasId": 123,
  "action": "start" | "move" | "end" | "clear",
  "points": [{ "x": 100, "y": 200 }, ...], // для move может быть массив точек
  "color": "#FF0000",
  "strokeWidth": 5,
  "senderId": 456
}
Сервер ретранслирует это событие всем остальным клиентам в той же комнате (кроме отправителя).

request-snapshot — если новый клиент подключается к существующему холсту, он может запросить текущее состояние. Можно реализовать через отдельное событие, но проще, чтобы при входе в холст клиент загружал последний сохранённый PNG через REST, а затем дорисовывал новые штрихи из истории действий (если она хранится). Однако для простоты можно не хранить историю на сервере, а полагаться на то, что при подключении к холсту клиент получит текущий PNG и будет получать только новые действия.

Комнаты: комната для пары (по couple_key) позволяет изолировать трафик.

3.4. Виджет на главном экране
Виджет трёх размеров (small, medium, large), аналогично другим разделам (настроение, активности).

Отображает последний рисунок, отправленный партнёром (берётся из thumbnail_url последнего обновлённого холста, где created_by != текущий пользователь). Или можно выбрать холст, помеченный как «для виджета».

Обновление виджета через FCM (Firebase Cloud Messaging) – на сервере уже есть функция sendPushToPartner, которую можно расширить для отправки события обновления виджета. Либо через периодический запрос к серверу (WorkManager).

При нажатии на виджет открывается редактор с соответствующим холстом.

3.5. Экран блокировки (Lock Screen) как холст
На экране блокировки появляется возможность рисования. Активация: через постоянное уведомление с кнопкой «Рисовать» (как в разделе «Расстояние»), либо через специальную настройку.

При активации открывается LockScreenDrawingActivity с флагами showWhenLocked и turnScreenOn. Интерфейс:

Весь экран — холст (как в редакторе).

Минимальная палитра (6 базовых цветов) появляется по свайпу снизу или долгому тапу.

Кнопка «Готово» (или жест) — сохраняет рисунок и закрывает режим, возвращая стандартный lock screen.

Рисование на lock screen синхронизируется с партнёром через WebSocket так же, как в обычном редакторе. При этом используется либо текущий активный холст (например, последний открытый), либо можно создать временный холст, который потом сохранится как новый.

Если партнёр не в режиме lock screen, он всё равно получит штрихи через WebSocket, и они отобразятся, если у него открыт соответствующий холст в приложении. Если холст не открыт, можно сохранить действия в памяти и применить при следующем открытии (но это сложно). Для простоты можно договориться, что lock screen работает с определённым холстом (например, «Быстрые зарисовки»).

4. Технические требования к клиентской части (Android)
Язык: Kotlin

UI: Jetpack Compose (с кастомной iOS-стилистикой)

Сеть: Retrofit для REST, Socket.IO для WebSocket (библиотека socket.io-client).

Хранение: Room для локального кэширования списка холстов и, возможно, действий (но для начала можно без этого).

Рисование: Кастомный Canvas composable, отслеживающий касания через Modifier.pointerInput. Хранение штрихов в ViewModel в виде списка DrawPath (путь + цвет + толщина).

Виджеты: Glance или классические AppWidget с обновлением через WorkManager (получение последнего thumbnail_url).

Lock screen: Activity с флагами окна и кастомным содержимым.
